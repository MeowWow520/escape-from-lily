## 代码规范

> **本规范为个人最佳实践，参考了JavaDoc和其他开发者的代码示例，并非完全按照官方建议的代码格式**

### 类
本文介绍代码格式总体上来说，本程序原代码符合以下几点：

- 1、类名和头文件名称**相等**。例如：`class Game` 类的头文件是 `Game.h` 。
- 2、头文件中**只包含**方法声明。
- 3、方法的**实现**应定义在 `.cpp` 文件中。
- 4、**类内不对成员变量初始化**，变量初始化在构造函数中实现。

### 变量命名
在方法的命名上使用[驼峰命名法](./)，私有成员变量应以 `_` 结尾

### 类内方法和变量的定义顺序
```cpp
class MyClass
{
    // 私有成员变量
    // 构造函数
    // 公有方法声明 继承
    // 私有成员 Getter 和 Setter
    // 公有方法声明 非继承
}
```
